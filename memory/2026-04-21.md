### Vercel 배포 이슈 및 해결
- 서비스: GitHub + Vercel 연동 (www.laundryops.co.kr)
- 문제1: `preview_images.js`(607KB)를 GitHub 웹 UI로 업로드 시 빈 파일(0 bytes)로 올라가는 문제
- 해결1: Git CLI로 직접 `dist/` 폴더에서 push 진행 (GitHub PAT 사용, `credential.helper store` 등록 완료)
- 문제2: 로그인 에러 (`window.login is not a function`) 발생
- 원인2: 
  1. `index.html`에 잔존하던 구버전 패치 스크립트(`patch_*.js`)들의 구문 오류(`Illegal return statement`)가 전체 스크립트 실행을 방해함.
  2. `index.html`과 `app_v38.js`에 `const LOGIN_HINTS`가 중복 선언되어 `SyntaxError: Identifier has already been declared` 발생하며 `app_v38.js` 실행 중단.
- 해결2: 
  - `index.html`에서 불필요한 구버전 패치 스크립트 및 `billing_logic.js` 제거 (이후 요금제 관련 오류로 인해 `billing_logic.js`만 다시 복구).
  - 중복 선언된 `LOGIN_HINTS`를 `window.LOGIN_HINTS`로 수정하여 호환성 문제 해결.
  - 브라우저 캐시 문제를 우회하기 위해 `app_v38.js?v=timestamp` 방식으로 버전 쿼리스트링 자동 업데이트.
  - Vercel 강제 재배포를 위해 Empty commit 후 push 진행.
